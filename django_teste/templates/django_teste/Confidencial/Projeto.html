Por que fazer o projeto?

Procuramos evitar possíveis furtos por parte dos cobradores e poupar tempo e dinheiro, uma vez que não mais teria que contratar alguém para conferir manualmente os tickets e o valor obtido com suas vendas.



Resumo

Nós pretendemos melhorar o sistema de cobrança dos transportes de viagens intermunicipais. Iremos fazer uma máquina que irá registrar todas as vendas de passagens, assim ajudando a descobrir se houve furto sobre o valor total de vendas. Além disso irá ter uma certa economia, pois não será necessário contratar um funcionário para conferir a quantidade de tickets e saber quantas passagens foram vendidas. Isso proporcionará uma maior ‘velocidade’ neste processo de verificação, uma vez que quando for registrado pela máquina (a venda de uma passagem) imediatamente, será enviada para o banco de dados do site. Não temos um preço estimado, mas pode sair não tão barato.



Objetivos

Gerais: Aumentar os lucros das empresas de ônibus de viagens.

Específicos: Modelar/obter uma máquina (ou um aplicativo) que automatize o processo de venda de tickets; Fazer um site utilizando banco de dados para armazenar os dados das vendas; Encontrar um meio de certificar que todas as pessoas que entraram e saíram do ônibus compraram as passagens.



Metodologia

Iremos fazer um site utilizando HTML, o qual conterá um banco de dados. Além disso, pretendemos fazer uma máquina que irá registrar as cobranças e enviar os dados para este site. Vamos utilizar Python para a programação.
